Reverse Coding — ExeBit
User Guide

Read carefully before the contest

Quick Overview

  • You will connect to a remote reverse coding server.
  • The server provides input-output behavior.
  • Your task is to deduce the hidden logic.
  • Implement the solution and submit on Codeforces.

§0 Before You Begin

  • Ensure you have a working terminal and internet connection.
  • Login to your Codeforces account.
  • Keep the contest page open.
  • Join the WhatsApp Group for queries and updates.
  • Server IP and port will be shared during the contest.

§1 Connecting to Server

Linux / Mac (Recommended)

nc <ip> <port>

If Mac netcat gives issues on your system, use the Google Colab option below.

Mac Alternative (Google Colab)

  1. Open Google Colab and create a new notebook.
  2. Paste and run these cells in order.

Cell 1 — Install netcat

 # Run this cell only once
!apt-get update -y
!apt-get install -y netcat-openbsd

Cell 2 — Connect to contest server

!nc <ip> <port>

Replace <ip> and <port> with values shared during the contest.

Windows (Telnet Setup)

  1. Open Command Prompt as Administrator
  2. Enable Telnet:
dism /online /Enable-Feature /FeatureName:TelnetClient
  1. Verify installation:
telnet
  1. Connect to server:
telnet <ip> <port>

Example:

telnet 35.192.44.62 6060
The actual IP and port will be provided during the contest.

§2 Login Process

After connecting, you will be prompted:


Enter name:
Enter Codeforces username:
Enter password:
  • Enter your name
  • Enter your Codeforces handle
  • Enter the password shared during contest

On successful login, your session begins.

§4 How to Solve

  • Provide custom inputs to the server
  • Observe corresponding outputs
  • Identify patterns and logic
  • Test multiple cases before coding

Example:

Input:
1 2 3

Output:
6

§5 Submission

  • Write your solution in your preferred language
  • Submit it on Codeforces
  • Follow exact input/output format specified in the problem
⚠️ Important: The server does NOT accept submissions. It is only for discovering problem logic.

§6 Important Notes

  • Each connection = one session
  • If you exit, you must reconnect
  • Your Codeforces handle is used for identification
  • Avoid random brute force — think patterns
